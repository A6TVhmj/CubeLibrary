# CubeLibrary (CL)

**CubeLibrary**（简称 **cl**），是一个向 Herbert Kociemba 经典的 Delphi 版 Cube Explorer 致敬的现代化 Python 重制版。

本项目利用 `ttkbootstrap` 赋予了原版等距视角魔方编辑器全新的现代 UI，并在底层用纯 Python 从零重构了两阶段算法。经过架构升级，目前的 CL 已从单一的求解器，进化为集专业测速、公式编译、残缺推导与最少步分析于一体的桌面端魔方工作站。

## ✨ 核心特性

* 🎨 **交互式展开图**：还原经典的 3D 展开图视觉体验。支持基础填色与自动推导面色，配合快捷键（Ctrl 忽略整块、Shift 忽略朝向）可快速设定未知状态。
* ⏱️ **专业训练计时器**：内置符合 WCA 标准的测速模块。支持 `+2` 与 `DNF` 惩罚切换、自动统计 Ao5 与 Ao12，并支持历史记录的动态回顾与修改。
* 🔄 **数据联动复盘**：打破组件壁垒，在计时器中遇到特定打乱时，可一键将其状态同步至编辑器，配合底层算法进行极限十字或最优解分析。
* 🧠 **高级公式解析**：公式输入框内置递归解析器，原生支持复杂魔方语法，包括交换子 `[R, U]`、共轭 `F: [R, U]`、循环展开 `(R U)3` 以及整体逆运算。
* 🚀 **双引擎求解**：
  * **两阶段极速模式**：快速搜索，瞬间给出还原公式。
  * **最少步最优模式**：基于迭代加深搜索，确保输出绝对最短路径，支持次优解的无尽搜索。
* 🧩 **残缺状态推导**：支持输入带有未知色块或仅确定颜色但不确定朝向的魔方状态。引擎内置角/棱块排列穷举与奇偶校验，自动输出所有物理合法的补全方案。
* 🌍 **多语言与主题**：支持简体中文、繁体中文、英文与波兰语的热切换；内置多种界面主题，一键切换明暗模式。

## ⚡️ 底层架构

本项目拥有完全自主掌控的后台计算引擎，并实施了以下优化以保障纯 Python 环境下的运行效率：

1. **内存视图提速**：在核心的高频循环中，将百万量级的状态剪枝表转换为 Python 原生 `bytes` 内存视图，有效消除了多维数组的寻址与装箱开销，显著提升寻路速度。
2. **多维奇偶校验**：重构了残缺状态补全器，在遇到忽略朝向的色块时，将严格匹配降维为集合子集匹配，同时在 DFS 遍历中精准执行色向和与排列奇偶性的物理拦截。
3. **全局迭代加深**：最少步模式严格按照总步数逐级展开搜索树，避免了传统深搜可能造成的次优解抢占，保证结果列表顶端始终为数学上的最优解。
4. **异步队列通信**：采用并发队列平衡后台高强度算力与前台重绘，通过定制的异常机制实现对深层递归树的快速中断，确保高刷新率 UI 不卡顿。
